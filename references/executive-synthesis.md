## グレード決定ロジック

```
A（優秀）:
  - CRITICAL: 0件
  - HIGH: 0件
  - 全カテゴリ平均★4以上
  - 技術的負債トレンド: 安定または減少

B（良好）:
  - CRITICAL: 0件
  - HIGH: 1-3件
  - 全カテゴリ平均★3.5以上
  - 技術的負債トレンド: 問わない

C（改善必要）:
  - CRITICAL: 0-1件
  - HIGH: 4-8件
  または CRITICAL 1件

D（大幅改善必要）:
  - CRITICAL: 2件以上
  または HIGH: 9件以上
  または フラジリティスコア40超のファイルが3以上

E（重大問題）:
  - CRITICAL: 3件以上
  または セキュリティ CRITICAL が存在
  または データ消失リスクがある
```

## 良いパターン抽出ガイド

`.claude/patterns.md` への追加候補として以下を特定する:

```
抽出すべきパターンの条件:
1. このコードベースが特に上手くやっていること
2. 他のプロジェクトにも適用できる汎用性があること
3. 非自明（自然にたどり着くのが難しい）こと

例:
- エラーハンドリングの一貫したパターン
- 型安全なAPI呼び出しの方法
- テストの書き方のパターン
- 設定管理の仕組み
- 認証フローの実装パターン

出力形式:
### 発見されたパターン: [パターン名]
場所: [ファイル:行番号]
内容: [パターンの説明]
推奨理由: [なぜ良いパターンか]
patterns.md への追記案: [コードスニペット]
```

## スコアの計算方法

各視点の★評価基準:

```
★★★★★: 問題なし、模範的な実装
★★★★☆: 軽微な改善点のみ（LOW問題）
★★★☆☆: 改善推奨（MEDIUM問題あり）
★★☆☆☆: 改善必要（HIGH問題あり）
★☆☆☆☆: 即座に対応必要（CRITICAL問題あり）
N/A:    プロジェクトタイプ上、評価対象外
```

カテゴリ平均の計算: N/Aを除いた視点の★の平均値（5段階）
